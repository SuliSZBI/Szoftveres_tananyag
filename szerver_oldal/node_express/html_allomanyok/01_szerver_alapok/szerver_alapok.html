Szerver alapok

Ebben a részben összefoglaljuk, hogy milyen kezdeti műveleteket kell végrehajtani egy webszerverhez szükséges projekt létrehozásakor az express.js keretrendszer segítségével.

  • A MongoDB felületen hozzunk létre egy projektet Webshop néven. Ezen belül alkossuk meg a Konyvesbolt nevű adatbázisunkat.
  • Regisztráljunk a Cloudinary felületen, ha még nem lenne fiókunk.
  • Hozzunk létre egy Konyvesbolt nevű mappát egy kiválasztott helyen a számítógépünkön.
  • Ezen belül alkossuk meg a backend nevű mappát a szükséges állományok tárolására.
  • Nyissuk meg ebben a backend mappában a GitBash felületet, vagy húzzuk be a Visual Studio Code felületére és ott nyissunk egy terminált.
  • Adjunk ki a következő utasítást a GitBash felületen:
    npm init -y
    Ez felépíti a backend mappában a szükséges node környezetet.
  • Módosítsuk a kapott package.json állományt. A package-lock.json állományhoz ne nyúljunk! és
    1. "description": "Könyvesbolt szerveroldal" - a projekt leírása.
    2. "main": "./src/server.mjs" - az "indító" állomány.
    3. "scripts": { "start": "node ./src/server.mjs" } - a projektet indító script.
    4. "keywords": [ "express", "mongodb" ] - a projektet leíró kulcsszavak.
    5. "author": "John Doe" - a projekt szerzője.
    6. "type": "module" - utalás, hogy ES6-t használunk CommonJS helyett.
  • Telepítsük a szükséges npm csomagokat. Ehhez adjunk ki a következő utasítást a GitBash felületen:
    npm install bcrypt cloudinary cors dotenv ejs express mongoose
  • A módosult package.json állomány. és
    1. bcrypt - adatok titkosításához.
    2. cloudinary - képek tárolására alkalmas felhőalkalmazás kezeléséhez.
    3. cors - CORS kérések kezeléséhez.
    4. dotenv - környezeti változók kezeléséhez.
    5. ejs - nézetmotor (view engine) kezeléséhez.
    6. express - szerver alkalmazás létrehozásához.
    7. mongoose - MongoDB kezeléséhez.
  • Alakítsuk ki a szükséges mappastruktúrát a projekthez. A mappastruktúra felépítése.
    1. controllers - http metódus-ok kezeléséhez.
    2. middlewares - a http request és http response közötti műveletekhez szükséges függvények kezeléséhez.
    3. models - MongoDB modellek kezeléséhez.
    4. public - statikus állományok kezeléséhez
    5. routes - http metódus-ok csoportosításához.
    6. src - szerver működéséhez szükséges állományok csoportosításához.
    7. src/routing - a route-okhoz szükséges állományok csoportosításához.
    8. utils - egyéb hasznos dolgokhoz (pl. adatbáziscsatlakozás) szükséges függvények kezeléséhez.
    9. views - a szerveroldali nézetek kezeléséhez.
  • Hozzuk létre a .env és .gitignore állományokat. , , és
    1. .env - környezeti változókat (pl. PORT számok, API-kulcsok stb.) tartalmazó állomány.
    2. .gitignore - mit ne töltsünk fel a GitHub-ra.
